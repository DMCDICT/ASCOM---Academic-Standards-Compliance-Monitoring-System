CSS Loading Order Fix

The following changes have been made to fix modal styling inconsistencies:

✅ Fixed: CSS loading order in content.php
  • modal-add-user.css now loads BEFORE modals.css
  • This ensures specific modal styles take precedence over generic ones
✅ Fixed: Generic form styles in global.css
  • Added :not() selectors to exclude #addUserModal, #addDepartmentModal, #userDetailsModal
  • This prevents generic styles from overriding specific modal styles
✅ Fixed: Generic form styles in school-calendar.css
  • Made .form-group, .form-actions, and button styles specific to #addTermModal and #addSchoolYearModal
  • This prevents calendar styles from affecting the user creation modal
✅ Fixed: Conflicting styles in modals.css
  • Removed conflicting #addUserModal styles that were overriding modal-add-user.css
  • These styles are now handled exclusively by modal-add-user.css
